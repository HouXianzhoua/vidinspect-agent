# 数据质检规范汇总

下表汇总了数据质检的各项检查规范、检测方式及备注。

检测方式图例：🟢 已实现/代码可检测 ｜ 🟡 待定 ｜ 🔴 无法检测

| 序号 | 质检项 | 详细描述 | 检测方式 | 备注 |
| --- | --- | --- | --- | --- |
| 1 | 二次抓取 | 视频中不应出现多次抓取同一物体的情况。每次抓取都应该是单次且有效的| 🟢 多模态（regrasp，已实现） | 抽帧→模型逐帧逐夹爪判 side/holding→代码侧按机械臂去抖后统计该臂持有段数，某臂 ≥2 段即命中（每臂单目标，双臂各抓一次正常）；默认 WARN |
| 2.1 | 动作不流畅 | 视频中不应出现同一动作尝试多次的情况，应保持连贯顺畅。| 🟡 无法检测 | |
| 2.2 | 机械臂异常-卡顿 | | 🟢 代码检测（jump 主判，dup_frame/freeze 兜底） | 2 的子类别；jump 抓卡顿后的突跳/瞬移，周期性微卡由 dup_frame、单段长冻结由 freeze 覆盖 |
| 3 | 开始与结束归位停留时间长 | 视频开头或结尾有超过 2s 的静止 | 🟢 代码检测（endpoint_static） | |
| 4 | 未按文档步骤采集 | 背景道具与文档描述 90% 符合即可；操作步骤需与任务描述一致，但可不按顺序，如视频中先做第一步、再做第三步也是可以的 | 🟡 待定 | ？ |
| 5 | 视频画面卡死 | 视频中出现长时间卡帧 | 🟢 代码检测（freeze） | |
| 6 | 视频画面掉帧 | 多段短时卡帧与掉帧，视频像在慢放 | 🟢 代码检测（dup_frame，已覆盖） | |
| 7 | joint 值异常 | | 🟡 待定 |？ |
| 8 | 触碰多余 | 视频中不应出现不必要的触碰或干扰物体的动作，所有操作应直接且有目的性（主要针对非目标物体的碰触，如物品摆放密集则忽略此项） |🔴 无法检测| |
| 9 | 量产数据 | | 🟡 待定 |？ |
| 10 | 标定点问题 | | 🟡 待定 |？ |
| 11 | 噪点问题 | 视频有严重噪点（含**灯光太暗导致的弱光增益/高 ISO 噪声**，已合并序号 22） | 🟢 代码检测（noise） | 噪声方差估计覆盖弱光增益噪声；注意本项只判噪声强度，不判画面整体亮度（欠曝本身见序号 20.4 brightness） |
| 12 | 夹爪出境 | 出镜持续 1s+ | 🟢 gemini（gripper_offscreen，已实现） | 抽帧→Gemini 逐帧判定夹爪是否在画面内→代码算最长连续出镜时长 |
| 13 | 任务名称与任务描述 | | 🟡 待定 | ？|
| 14 | 根据需求多样性 | | 🟡 待定 | ？|
| 15 | 遮挡操作物品问题 | 视频首帧存在夹爪遮挡操作物品 | 🟡 待定 | |
| 16 | 夹取位置过于极限问题 |抓取的是物体边缘而不是主体 | 🟡 待定 |  |
| 17 | 含桌布场景，杜绝操作时误夹桌布 | | 🟡 待定 |  |
| 18 | 画面保持一致 | 不要出现画面机械臂在动，夹爪画面不动的情况| 🟡 ai |？ |
| 19 | 桌面操作物颜色相同问题 | 操作物品与桌面颜色相同，很难分辨出操作物品的位置和大小| 🟢 多模态（colormatch，已实现） | 静态属性，无需时序：本地少量抽帧→模型逐帧判「被操作物体与桌面是否同色难分辨」→代码侧在可识别物体的帧里按「难分辨」占比 ≥ 阈值判定；默认 WARN |
| 20.1 | 相机画面-黑屏 | 画面接近全黑 / 无信号 | 🟢 已覆盖（visual blackdetect） | 不另设检测器，见序号 14（视频画面卡死无关，黑屏由 visual 判黑屏时长比） |
| 20.2 | 相机画面-花屏 | 画面雪花 / 条纹 / 马赛克 / 撕裂 | 🟢 部分覆盖（integrity） | 解码报错型花屏由 integrity 全解码捕获；能正常解码的传感器/传输层花屏暂未覆盖 |
| 20.3 | 相机画面-偏色/白平衡异常 | 画面严重偏色、白平衡异常 | 🔴 暂不实现 | 纯色彩判断真实场景误报率高（暖光、彩色桌布、单色主体），先不做 |
| 20.4 | 相机画面-画面过暗/欠曝 | 画面整体偏暗但非全黑 | 🟢 代码检测（brightness） | 全画面平均亮度中位数 < 阈值即命中；与噪点（序号 11 只判噪声强度）、黑屏（visual）互补；默认 WARN |
| 21 | 物体滑落问题 | 夹爪夹住以后，物体滑落 | 🟢 多模态（object_slip，已实现） | 与 1 二次抓取共用架构但判据不同（非其子集）：抽帧→模型逐帧逐夹爪判 holding/gripper_closed→代码侧逐臂去抖后，看"持有结束时夹爪是否仍闭合"——仍闭合却脱手即滑落，张开放下为正常；默认 WARN |
| 22 | 灯光太暗，导致噪声严重 | 噪声部分等价于序号 11 | 🟢 已合并入序号 11 | 弱光增益噪声由序号 11（noise）覆盖；"画面过暗/欠曝"本身已由序号 20.4（brightness）实现 |
| 23 | 抖动，视频抖动晃动严重 | | 🟡 待定 | 天轶暂时无法避免，尽量别太严重 |
